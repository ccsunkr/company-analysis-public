# 기업분석 · 정량 밸류에이션 (공개용 정적 사이트)

DART 공시 기반으로 기업의 5년+ 재무 추세를 정리하고, **멀티플(PER·PBR) 밸류에이션**으로
"현재 기업가치 대비 주가"가 저평가/적정/고평가인지 판단하는 웹페이지입니다.
체리형부 「기업 분석 글 작성 가이드」 방법론(DART → 분기 재무 → 조합 차트 → 리스크 헤지 해설)을 웹으로 옮긴 것입니다.

의존성 0 · 서버 불필요 · GitHub Pages에 그대로 올려 공개할 수 있습니다.

## 구성

```
company_analysis/
├─ index.html          기업 목록(카드) + 상세 분석
├─ editor.html         기업 추가/편집 (실시간 밸류에이션·차트 미리보기)
├─ README.md
└─ assets/
   ├─ css/style.css
   └─ js/
      ├─ data.js       ★ 기업 데이터 원본 (커밋되는 source of truth)
      ├─ valuation.js  밸류에이션·추세 계산 (순수 함수)
      ├─ charts.js     의존성 없는 SVG 조합 차트
      ├─ app.js        목록/상세 렌더링
      └─ editor.js     편집기
```

## 로컬에서 보기

- `index.html`을 브라우저로 그냥 열어도 동작합니다(데이터는 `.js`로 로드하므로 서버 불필요).
- 편집기에서 「로컬 저장」한 기업은 그 브라우저(localStorage)에만 저장됩니다.

## 기업 추가/편집 → 공개 반영 워크플로

1. `editor.html`에서 DART 재무·시세를 입력합니다.
   - **가장 빠른 방법: OpenDART API 자동 수집** — 종목코드 입력 → 「⚡ OpenDART에서 재무 자동 수집」 버튼 한 번으로
     분기 매출액·영업이익·순이익 + 자본총계 + 발행주식총수가 채워집니다(아래 OpenDART 절 참고).
   - 분기 실적(매출·영업이익)은 필수. 매출원가/판관비/영업현금흐름/순이익은 선택(노란 칸).
   - 밸류에이션: 현재 주가, 발행주식수, 순이익 TTM(또는 분기 순이익 4개), 자본총계, 과거 PER/PBR 밴드.
   - 오른쪽 미리보기에서 시그널·상승여력·조합 차트를 즉시 확인.
2. **「💾 로컬 저장」** — 이 브라우저에서 바로 목록에 표시(임시).
3. **「⬇ data.js 전체 (커밋용)」** — 모든 기업이 담긴 `data.js`를 내려받습니다.
4. 내려받은 파일로 `assets/js/data.js`를 **교체하고 커밋 → push**. 공개 사이트에 영구 반영됩니다.

> 개별 기업만 백업하려면 「⬇ 이 기업 JSON」으로 내보내고, 나중에 「⬆ JSON 가져오기」로 복원할 수 있습니다.

## 밸류에이션 방법 (멀티플 + 역사적 밴드)

기본 지표:
- 시가총액 = 현재 주가 × 발행주식수
- **EPS(TTM)** = 순이익TTM ÷ 발행주식수 → **현재 PER** = 주가 ÷ EPS  (순이익TTM = 최근 4개 분기 순이익 합)
- **BPS** = 자본총계 ÷ 발행주식수 → **현재 PBR** = 주가 ÷ BPS

### 적정주가는 어떤 기준으로 잡나 (핵심)
적정주가는 **"이 기업의 이익·자본에 '정상적인 배수'를 곱하면 주가가 얼마여야 하는가"** 로 계산합니다.

1. **목표 배수(정상 배수)를 정한다** — 우선순위:
   ① 사용자가 직접 입력한 목표 PER/PBR → ② 수동 입력한 과거 밴드의 평균 → ③ **역사적 PER/PBR 시계열의 중앙값(median)**.
   즉 별도로 지정하지 않으면 **그 종목이 과거에 거래돼 온 PER/PBR의 중앙값**을 "정상 배수"로 씁니다(그 기업 고유의 눈높이).
2. **세 방식으로 적정주가를 각각 구한다**:
   - 자본 기준: **적정주가₁ = BPS × 목표 PBR**
   - 배당 기준: **적정주가₂ = DPS(주당배당금) ÷ 목표 배당수익률** (DPS 입력 시)
   - 이익 기준: **적정주가₃ = EPS(TTM) × 목표 PER**
3. **적정주가 밴드 = [ 앵커들의 최소 ~ 최대 ]**, 대표 **적정주가 = 가중 평균: PBR 0.5 · 배당 0.3 · PER 0.2**
   (없는 앵커는 가중치를 남은 앵커에 재배분).
   **기업별 프리셋**으로 조정 가능(편집기 「적정주가 앵커 가중 프리셋」): 이익이 꾸준한 **안정 성장형**은 PER 0.5·배당 0.3·PBR 0.2,
   **경기민감·자산형**(반도체 등 사이클 기업)은 PBR 0.6·배당 0.3·PER 0.1.
   사이클 기업의 이익 고점에서 낮아 보이는 PER은 착시(고점 매수 함정)이므로 PER 비중을 오히려 낮춥니다.
4. **상승여력 = 적정주가(가중평균) ÷ 현재 주가 − 1**. **12개월 관점**의 신호입니다.

> **가중치의 근거** — KOSPI top30 대상 12개월(T-12) 표본외 검증(사용자 논문, 「KOSPI 예측지표 탐색」 2026):
> 자기 역사 대비 **PBR**이 가장 강건(표본외 R² 중앙값 +10.8%, 양(+) 기업 70%), **배당수익률**은 약한 양(+2.1%),
> **이익수익률(1/PER)**은 T-12에서 실패(−3.4%). 그래서 PBR > 배당 > PER 순으로 가중합니다.
> 같은 이유로 저평가/고평가 **시그널 투표에서도 PBR 밴드 위치를 2배 가중**합니다.
> 단, 이 신호는 "그 종목이 자기 과거 대비 쌀 때 이후 1년에 걸쳐 회복한다"는 **종목 내 시계열** 신호이며,
> "싼 종목이 비싼 종목을 이긴다"(종목 간 비교)는 백테스트에서 확인되지 않았습니다. 6개월 이하 단기 타이밍에도 무력합니다.

### 판정(시그널)
두 신호를 종합해 **저평가 / 적정 / 고평가**:
- **상승여력**: +15% 이상 → 저평가 신호, −15% 이하 → 고평가 신호, 그 사이 → 중립
- **현재 PER의 밴드 내 위치**: 역사적 밴드의 하위 25%(P25) 이하 → 저평가, 상위 25%(P75) 이상 → 고평가
- 두 신호의 평균으로 최종 판정. (역사적 밴드가 있으면 백분위 P25·중앙·P75를 쓰고, 없으면 수동 밴드의 하단/평균/상단 사용)

### 주의·한계
- 목표 배수를 **과거 중앙값**으로 쓰므로, 구조적 성장/쇠퇴로 적정 배수 자체가 변한 경우에는 과대·과소평가될 수 있습니다(가정에 민감).
- 발행주식수·순이익이 없어 적정주가를 못 구해도, **현재 PER의 역사적 밴드 내 위치**만으로 저평가/고평가를 판단합니다(이때 현재 PER은 시계열 최신값으로 대체).
- DCF·부문가치·순현금 등은 반영하지 않는 **단순 멀티플 모델**입니다. 참고용이며 매수·매도 추천이 아닙니다.

## 투자 원칙 체크 — 주가 = 숫자 × 촉매

사용자 투자 원칙을 사이트에 내장한 기능입니다.

- **비교표** (메인 상단 「카드↔비교표」 토글): 등록 종목만으로 **같은 업종끼리** 비교. 핵심 3지표(PER·ROE·영업이익률)+**포워드PER** 중심,
  부채비율·배당·매출/영익 YoY·3대 스크리닝·7대 필터 점수·촉매·신호 열. 열 클릭 정렬, 그룹 내 최저 포워드PER·최고 ROE 강조,
  재고/채권이 매출보다 빨리 늘면 ⚠.
- **숫자 × 촉매 신호** (상세 페이지 최상단): 숫자 = 3대 스크리닝(매출↑+영업이익↑+흑자, 최근 분기 YoY) 자동 판정 ×
  촉매 = 유효 촉매(6~12개월 시한, 시한 경과 시 만료) 존재. 곱셈이라 하나라도 0이면 경고 문구(가치함정/폭탄).
- **7대 필터 (7점)**: ①실적 지속성(영업이익 YoY 3분기+ 연속) ③재무 안정성(부채비율<100%·OCF+) ④마진 개선(OPM 4Q vs 직전4Q)
  ⑤동종(내 종목) 대비 포워드PER — 자동 / ②이익의 질 ⑥촉매 ⑦주봉 바닥권 — 편집기 체크.
- **밸류 3좌표**: 포워드 PER(내년 예상 순이익 입력) 기준 저(10)–중(20)–고(60) 게이지. "저→중 재평가 구간" 전략 표기.
- **P/Q 사이클 유형**: P-사이클 지정 시 "정점의 저PER은 함정" 경고 + PBR 중심 프리셋 권장.
- 입력(편집기 「🎯 투자 원칙 체크」): 촉매(내용·시한·근거등급 Fact/A·Fact/B·추정·판단·상태), P/Q 유형, 수기 확인 2종.
  공유 저장 시 친구들도 서로의 촉매·판단을 보고 조언할 수 있습니다.
- **🌐 거시 매도 트리거 (AI 자금 사슬) — 별도 카드, 모든 종목 공통**: 메인 상단 카드에서 stockanalysis.com 분기 실적으로 **자동 판정**.
  ①② 하이퍼스케일러(MSFT·GOOGL·AMZN·META) 현금창출력·Capex → ③ TSM(파운드리) → ④⑤ NVDA 매출·OPM.
  판정 규칙: ① OCF TTM YoY<0 **또는** (FCF<0 **그리고** Capex<0) / ② 합산 Capex TTM YoY<0 / ③ TSM 매출 TTM YoY<0 /
  ④ NVDA 매출 QoQ<0(순차 감소) / ⑤ NVDA OPM 전분기 대비 하락.
  > ⚠ **왜 ①이 단순 FCF가 아닌가**: Capex가 급증하면 FCF는 기계적으로 줄어듭니다(= 상류 호황인데 매도 신호로 오독).
  > 그래서 돈줄 자체인 **OCF**로 판정하고 FCF는 참고로 표시합니다. (실측: OCF +30% · Capex +70% · FCF −16% → 점등 안 함)
  각 항목은 **수동 지정**(가이던스로 미리 알 때) 가능하며, ①② 점등 시 **모든 페이지 상단에 거시 경보 배너**가 뜹니다.
  데이터는 6시간 캐시. 프록시(워커 `?sa=` — 최신 코드 재배포 필요) 경유.
- **매도 트리거 5단계 (종목별 자금 사슬)**: 업종마다 상류→하류 사슬이 다르므로 **종목마다** 5단계를 둡니다.
  기본값(전방 FCF→Capex→OEM/ODM→대장주 매출→대장주 OPM)을 편집기에서 업종에 맞게 수정(예: 화장품 = 중국 소비·면세점…).
  상세 페이지에서 각 단계를 클릭해 점등/소등하고, **①② 점등 시 그 종목 상세 상단에 빨간 "비중 축소" 경보 배너**(⑤까지 기다리면 늦음),
  하류만 점등 시 노란 경보. 목록 카드에도 🔻 배지. 공유본이면 친구들과 상태·메모를 함께 관리(점등 시 공유 저장), 예시/로컬본이면 이 브라우저에 저장.

## 성장 · 수익성 핵심 지표 (가치투자 체크)

상세 페이지 「성장 · 수익성 핵심 지표」 섹션과 목록 카드에 자동 계산·표시됩니다.

| 지표 | 계산 | 판단 기준(색상) |
|---|---|---|
| **ROE** | 순이익(TTM) ÷ 자본총계 | ≥ 15% 녹색(우량) · < 5% 적색 |
| **EPS 성장률** | ① 직접 입력한 예상 성장률(%) → ② 연간 순이익 CAGR(4분기 다 있는 완전 연도, 최근 3년) → ③ TTM YoY | ≥ 10% 녹색 · 음수 적색 |
| **PEG** | 현재 PER ÷ EPS 성장률(%) | **≤ 1 녹색(성장 대비 저평가)** · ≥ 2 적색 |
| **매출·영업이익·순이익 성장률** | 완전 연도 기준 3년 CAGR + TTM YoY(최근 4분기 vs 직전 4분기) | — |

- EPS 성장률은 발행주식수가 크게 변하지 않았다는 가정에서 **순이익 성장률로 근사**합니다(대규모 증자·자사주 소각 기업은 예상 성장률을 직접 입력 권장).
- 연간 요약 표에는 **순이익·순이익 YoY** 열이 추가되고, 4분기가 안 채워진 연도는 `(nQ)`로 표시됩니다.
- 필요 데이터: 분기 **순이익**(EPS·ROE·PEG), **자본총계**(ROE·PBR), **발행주식수**(EPS) — 모두 OpenDART 자동 수집으로 채울 수 있습니다.

## OpenDART API 자동 수집 (권장 입력 경로)

편집기에서 OpenDART 인증키(무료: https://opendart.fss.or.kr)와 종목코드만 넣으면
**분기 매출액·영업이익·당기순이익(최대 10년) + 자본총계 + 발행주식총수 + 배당(주당배당금·목표 배당수익률)**을 버튼 한 번에 수집합니다.
배당은 최신 사업보고서의 「배당에 관한 사항」에서 보통주 **주당 현금배당금**과 **현금배당수익률(최대 3개년 → 평균을 목표 배당수익률로)**을 가져와
배당 기준 적정주가 앵커에 바로 연결됩니다. (배당 이력을 눈으로 확인하려면: 네이버 금융 종목 페이지, Seibro(seibro.or.kr) 주식→배당정보, 또는 DART 사업보고서)

1. **프록시 필요**: OpenDART는 CORS를 막고 있어 Cloudflare Worker 프록시를 경유합니다 → [`serverless/README.md`](serverless/README.md).
   기존 네이버 프록시를 이미 배포했다면 **워커 코드를 최신본으로 교체**만 하면 됩니다(같은 URL 사용).
2. 편집기 「⚡ 자동 입력 ① — OpenDART API」에 **인증키** 입력(브라우저에 저장) → 조회 기간 선택 → 수집 버튼.
3. 동작: 고유번호 목록(zip, 최초 1회 후 30일 캐시) → 연도×보고서별 주요계정 조회 → 누적 손익을 분기값으로 변환(Q4 = 연간 − 3분기 누적) → 기존 분기와 병합.
4. 연결(CFS) 우선, 없으면 별도(OFS)로 자동 폴백. 12월 결산 법인 기준.
5. OpenDART 주요계정에 없는 **매출원가·판관비·OCF·CAPEX·재고자산·매출채권**은 기존 PDF 업로드로 보완.
6. 남는 수기/반자동 항목: 현재 주가(네이버 버튼), 역사적 PER/PBR 밴드(네이버/KRX CSV), 해설.

## DART 보고서 자동 입력 + 네이버 시세

편집기(`editor.html`)에서 DART 분기·반기 보고서 PDF를 업로드하면 **연결 요약재무정보·손익계산서·현금흐름표**를 파싱해 자동 입력합니다(브라우저 내 pdf.js, 서버 불필요).

- **자동 추출(→ 회사 단위로 환산)**: 매출액·매출원가·판관비·영업이익·순이익(손익계산서 3개월=분기값), 재고자산·매출채권·자본총계(요약), OCF·CAPEX(=유형+무형자산 취득). **전년 동기 손익**도 함께 채움.
- **발행주식총수 자동 추출**: 자본금/주당이익 주석에서 보통주 발행주식총수(소각 제외)를 추출해 **주 단위 그대로**(환산 없이) 입력 → 네이버 추정치보다 정확. 미기재 보고서면 가중평균유통보통주식수로 폴백.
- **여러 보고서 한 번에 업로드**: 파일 선택 시 2~3년치 분기·반기·사업 보고서를 한꺼번에 고르면, **업로드 순서와 무관하게 연도·분기 순으로 자동 정렬**해 적용(사업보고서는 Q4 계산을 위해 마지막). 미리보기에 각 파일의 유형·기간과 **미추출 항목 경고**(예: "⚠ OCF·재고자산 미추출")를 표시.
- **분기/반기보고서 = 그 분기(+전년 동기)**. **사업(연간)보고서 = 각 연도 Q4 자동 계산**(Q4 = 연간 − Q1 − Q2 − Q3, 해당 연도 1~3분기가 있으면 손익·현금흐름까지 자동; 3개년 연간값이라 여러 해 Q4를 한 번에).
- **자본총계·발행주식수·업데이트일자 = 가장 최근 보고서 기준**(여러 보고서를 올려도 최신 분기말 시점값으로 반영).
- **현금흐름 누적 자동 보정**: 반기·3분기의 OCF·CAPEX는 연초 누적(YTD) → 직전 분기를 빼서 분기값으로 자동 변환(1분기부터 순서대로 업로드 시).
- **현재 주가**: 「📈 네이버에서 현재가·주식수」 버튼(종목코드 기준). 정적 사이트는 CORS로 네이버를 직접 못 부르므로 **서버리스 프록시(Cloudflare Worker)** 를 한 번 배포해 편집기의 "네이버 프록시 URL"에 넣어두면 안정적으로 조회됩니다 → [`serverless/README.md`](serverless/README.md). 미설정 시 공개 프록시로 시도하고 실패하면 수기 입력으로 폴백. (발행주식수는 DART 정확값이 있으면 네이버 추정치로 덮지 않음.)
- 편집기 오른쪽 **입력 상태 체크리스트**가 자동 채워진 항목과 남은 수기 항목을 실시간 표시.

### PDF 업로드로도 자동이 안 되는(수기 입력) 항목
1. **현재 주가** — 보고서엔 없음. 네이버 버튼 또는 수기. (발행주식총수는 DART PDF에서 자동.)
2. **역사적 PER/PBR(밴드용)** — 「📊 네이버에서 과거 PER/PBR 불러오기」(최근 3~4년 자동) 또는 **KRX PER/PBR CSV 업로드**(여러 개 동시 업로드 가능, 날짜·PER·PBR 열 자동 인식 → 분기말만 추출, 파일 간 자동 병합). 불러오면 **밴드(최저·중앙·최고)와 목표 PER/PBR이 밸류에이션 입력에 자동 반영**되어 시그널이 즉시 계산됨.

> **일부 셀이 비는 이유**: OCF·CAPEX·재고자산·매출채권은 그 분기의 **자기 보고서(분기·반기·사업)** 에서만 나옵니다. 어떤 분기가 손익만 차 있고 CF/재무상태가 비었다면, 그 분기는 **다른 해 보고서의 "전년 동기"로만 들어온 것**(전년 동기는 손익만 제공)입니다. 그 분기의 실제 보고서를 올리면 채워집니다.
> **Q4의 경우**: 재고자산·매출채권은 사업보고서의 **당기말 잔액이 그대로 Q4 값**으로 들어갑니다. Q4 OCF·CAPEX는 **연간 − (1~3분기 합)** 으로 계산하므로 **그 해 1~3분기 OCF·CAPEX가 모두 있어야** 채워집니다 — 비어 있다면 그 해 1분기·반기 보고서를 함께 올리세요.
3. **해설/코멘트·목표 배수** — 정성 판단, 수기.
4. **부문별/제품별·스프레드·가동률** — 요약재무에 없음(사업의 내용/주석), 현재 미지원.

### 5년(20분기)을 다 채우려면
연도별로 **1분기보고서 · 반기보고서 · 3분기보고서 · 사업보고서** 4개를 (가능하면 1분기→사업보고서 순서로) 올리면 Q1~Q4가 완성됩니다. 각 보고서가 전년 동기 손익도 채우므로 실제로는 더 촘촘하게 쌓입니다.

## 확장 데이터 모델 (선택 입력 → 있으면 해당 섹션 자동 표시)

| 분류 | 입력 | 자동 계산·표시 |
|---|---|---|
| 손익 | 매출액·영업이익 (필수), 매출원가·판관비·순이익 | GPM(매출총이익률), 판관비율, OPM, **마진 추세 차트** |
| 현금흐름 | OCF, CAPEX | **FCF = OCF − CAPEX**, 현금흐름 차트 |
| 효율성 | 재고자산, 매출채권 | 재고회전율(=매출원가/재고), 매출채권회전율(=매출/채권) |
| 계절성 | (분기 라벨) | 같은 분기 연도별 비교 차트 |
| 밸류에이션 | 역사적 분기 PER/PBR | 백분위 밴드 + 현재 위치 |
| 주주환원 | 분기별 자사주 매입액 | 매입 추세 차트, 누적 |

- **재무 단위**는 기업별로 선택(조원·억원·백만원·천원). 예) 삼성전자=조원, 미원상사=백만원.
- **연결/별도** 재무제표 기준을 표시. (현재는 한 기준을 저장; 두 기준 동시 저장은 향후 확장)
- 두 예시가 포함됩니다: **삼성전자**(조원, 멀티플 기본) / **미원상사**(백만원, 확장 지표 전체 + 역사적 PER 밴드 + 자사주).

> 아직 미포함(향후): 제품별 매출(내수/수출)·제품가×판매량(P×Q)·원재료 스프레드·가동률 등 소재기업 심층 지표.

## 친구와 공동 분석 (공유 저장소)

친구도 사이트에서 **직접 분석하고 저장**할 수 있습니다. 저장된 기업은 **모두의 목록·「편집할 기업」 드롭다운에 즉시 반영**됩니다.

- 저장소: Cloudflare 워커의 KV (설정 5분 — [`serverless/README.md`](serverless/README.md) 「공유 저장소 설정」).
- 접근: 팀 **공유 비밀번호** 하나로 보호. 친구는 사이트 접속 → 「🔑 공유 데이터 보기」 버튼(목록 페이지) 또는
  편집기 「공유 설정」에 비밀번호 입력(브라우저에 저장, 1회) → 끝. 파이썬·API 키 불필요.
- 편집기 저장 버튼: **「☁ 공유 저장 (모두에게)」** = 친구들 화면에도 반영 / 「💾 로컬 저장 (나만)」 = 이 브라우저만.
- 고유번호(corp_code) 맵도 공유 저장소에 자동 캐시되어, 친구는 느린 zip 다운로드 없이 OpenDART 수집을 시도할 수 있습니다.
  (단, OpenDART가 해외 IP를 차단하면 워커 경유 재무 수집은 실패할 수 있음 — 그 경우 네이버 시세·수기 입력은 정상 동작하고,
  재무 자동 수집은 로컬 프록시를 쓰는 사람이 하면 됩니다. 공유 저장으로 결과는 모두에게 전달.)

### 정적 배포 (GitHub Pages)

사이트 파일 자체는 **순수 정적**이라 GitHub Pages에 올리면 됩니다.

**최초 1회 설정** (이 폴더는 이미 git 저장소로 초기화·커밋되어 있음):

1. **공유 저장소 설정** 후 `assets/js/config.js`의 `sharedUrl`에 워커 주소 입력.
   기존 로컬 저장 데이터가 있으면 편집기에서 불러와 「☁ 공유 저장」으로 공유 저장소에 올리기.
   (또는 「⬇ data.js 전체 (커밋용)」로 `assets/js/data.js`를 교체하는 정적 방식도 가능.)
2. github.com에서 새 저장소 만들기 (예: `company-analysis`, **Public** — 무료 Pages는 공개 저장소만 가능).
3. 이 폴더에서:
   ```bash
   git add -A && git commit -m "데이터 갱신"
   git remote add origin https://github.com/<아이디>/company-analysis.git
   git push -u origin main
   ```
4. 저장소 **Settings → Pages → Source: Deploy from a branch → main / (root)** 선택.
5. 몇 분 뒤 `https://<아이디>.github.io/company-analysis/` 가 생김 → 이 URL을 친구에게 공유.

**이후 업데이트**: 편집기에서 데이터 수집·수정 → 「⬇ data.js 전체」로 교체 → `git add -A && git commit -m "..." && git push` 만 반복.

> ⚠ **공개 범위 주의**: GitHub Pages URL은 링크를 아는 사람은 누구나 볼 수 있습니다(검색엔진 노출 가능성도 있음).
> 분석 내용을 완전히 비공개로 두고 싶으면 Cloudflare Pages + Access(이메일 인증, 무료) 같은 방식을 검토하세요.
> 친구가 보는 페이지에서도 「주식 성격」 드롭다운으로 가중 프리셋을 바꿔볼 수 있으며, 그 선택은 각자의 브라우저에만 저장됩니다(원본 data.js 불변).
