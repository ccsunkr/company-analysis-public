기본 정보
⚡ 자동 입력 ① — OpenDART API (권장)

위에서 종목코드·재무 단위·연결/별도를 설정한 뒤 버튼 하나로 분기 재무 전 항목(매출액·매출원가·판관비·영업이익·순이익·OCF·CAPEX·재고자산·매출채권 + 자본/부채총계·발행주식총수·배당)을 자동 수집합니다. → ROE·EPS성장률·PEG·PER·GPM·FCF·회전율이 바로 계산됩니다. API 키는 opendart.fss.or.kr에서 무료 발급.

매출액·매출원가·판관비·영업이익·순이익·OCF·CAPEX·재고자산·매출채권·자본/부채총계·발행주식수·배당까지 전 항목 자동 수집됩니다(전체 재무제표 API).

⚡ 자동 입력 ② — DART 보고서 PDF · 네이버 시세

① 위에서 재무 단위와 종목코드를 먼저 설정 → ② DART 분기·반기·사업 보고서를 여러 개 한꺼번에 선택해 업로드(연결재무제표 포함본) → ③ 네이버에서 현재가. 업로드 순서와 무관하게 연도·분기 순으로 자동 정렬해 적용하므로(사업보고서는 Q4 계산을 위해 맨 마지막), 2~3년치를 한 번에 올리면 됩니다.

OpenDART는 해외 IP(Cloudflare 워커)를 차단하는 경우가 많습니다. 수집이 시간 초과로 실패하면 터미널에서 python serverless/dart-local-proxy.py 실행 후 여기에 http://127.0.0.1:8321을 입력하세요(내 PC에서 직접 중계, 편집할 때만 켜두면 됨). Cloudflare Worker 배포 방법: serverless/README.md.

☁ 공유 설정 (친구들과 실시간 공유)

공유 저장·공유 목록은 Cloudflare 워커의 KV 저장소를 사용합니다. 워커에 KV(STORE)와 비밀번호(STORE_KEY)를 한 번 설정하면(serverless/README.md), 친구는 사이트에서 비밀번호만 입력하면 보고·분석·저장할 수 있습니다. 공유 서버 URL을 assets/js/config.js에 넣어 커밋하면 친구는 URL 입력도 필요 없습니다.

분기 실적 (선택한 재무 단위 기준)

노란 칸은 DART 사업/분기보고서에서 발췌하는 선택 입력입니다. 매출원가→GPM·판관비율, OCF·CAPEX→FCF, 재고자산·매출채권→회전율, 순이익(4분기)→TTM PER이 자동 계산됩니다.

분기매출액영업이익 매출원가판관비 순이익OCF CAPEX재고자산 매출채권
밸류에이션 입력 (멀티플)

적정주가는 세 앵커의 가중 평균입니다 (없는 앵커는 가중치 재배분). 기본 가중 근거: KOSPI top30 12개월 표본외 검증에서 자기 역사 대비 PBR이 가장 강건, 배당수익률 약한 양(+), 이익수익률(PER)은 실패. ⚠ 경기민감주의 사이클 고점에서 낮아 보이는 PER은 착시입니다 — 그런 기업은 PBR 중심 프리셋을 쓰세요.

역사적 PER / PBR (밴드 자동산출)

분기별 과거 PER(선택 PBR)을 입력하면 백분위(최저·P25·중앙·P75·최고) 밴드가 자동 계산되어 현재 위치를 판단합니다. 발행주식수·순이익을 안 넣어도 현재 PER은 이 시계열의 최신값으로 대체됩니다.

분기PER(배)PBR(배)

네이버는 최근 3~4년(연간+분기)만 제공 — 더 긴 밴드는 KRX 정보데이터시스템에서 PER/PBR CSV를 받아 업로드하세요. CSV는 날짜·PER·PBR 열을 자동 인식해 분기말 값만 추립니다.

🎯 투자 원칙 체크 — 숫자 × 촉매

주가 = 숫자 × 촉매(곱셈 — 하나라도 0이면 안 움직임). 숫자(3대 스크리닝: 매출↑·이익↑·흑자)는 분기 데이터에서 자동 판정되고, 여기에는 촉매와 수기 확인 항목을 입력합니다. 좋은 촉매: 구체적 · 검증 가능 · 6~12개월 시한 · 매출/이익 숫자로 연결. 막연한 테마는 촉매가 아님.
